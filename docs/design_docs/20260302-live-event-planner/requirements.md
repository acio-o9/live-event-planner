# Requirements: Live Event Planner

**Status**: Draft
**Created**: 2026-03-02
**Developer**: maru

## What（何を作るか）

社会人サークルで複数バンドが参加できるライブを企画・運営するシステム。

- Google アカウントを IdP とし、OIDCでログインできる
- ログインユーザーでバンドを構成できる
- バンドはライブごとにセットリストを所有する
- ライブイベントを企画すると大まかなマイルストーンが自動生成される

## Why（なぜ必要か）

- 企画・運営が属人化しており、担当者に依存した状態になっている
- スプレッドシートで管理しており、処理が煩雑でミスが起きやすい
- タスクがメンバーに偏らないようにしたい

## Requirements（要件）

### Functional Requirements（機能要件）

- [ ] Google IdP を利用したOIDCログイン処理の実装
- [ ] ライブ、バンド、ユーザーなどドメインモデルの設計と実装
- [ ] ライブ生成時のスケジュール（マイルストーン）の自動モデリング
- [ ] バンドメンバー管理（追加・変更）
- [ ] セットリスト管理（ライブ×バンド単位）

### Non-Functional Requirements（非機能要件）

- [ ] 初期ページ表示: 3秒以内
- [ ] テストカバレッジ: 80%以上
- [ ] モバイル・PC両対応のレスポンシブデザイン
- [ ] OIDC認証によるセキュアなアクセス制御（認証済みユーザーのみアクセス可能）

## Success Criteria（成功基準）

- [ ] Google OIDCログインが正常に動作する
- [ ] バンド・ライブ・セットリストのCRUD操作が全て動作する
- [ ] ライブ生成時にマイルストーンが自動生成される
- [ ] テストがすべて通過する（80%以上のカバレッジ）
