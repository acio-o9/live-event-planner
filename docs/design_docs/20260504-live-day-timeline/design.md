# Design: ライブ当日タイムライン

**Status**: Draft  
**Created**: 2026-05-04  
**Developer**: maru

---

## 概要

ライブ当日の細かいタイムスケジュールを管理するための縦型タイムラインビュー。  
バンドごとにレーンを持ち、逆リハ・本番・スタッフ共通タスクを一画面で把握できる。  
データはすべてインメモリ（揮発）。DB 永続化は現スコープ外。

---

## 画面構成

```
[ 叩きのタイムライン生成 ] [ クリア ] [ 一列表示 ]         凡例
[ バンドフィルター チップ群 ]

+--------+----------+----------+----------+----------+
| 時刻   | スタッフ  | Band 1   | Band 2   | ...      |  ← sticky header
+--------+----------+----------+----------+----------+
| 10:00  |          |          |          |          |
|        | [打ち合わ]|[リハーサル]|          |          |
| 11:00  |          |          |[リハーサル]|          |
| ...    |          |          |          |          |
| 14:00  |          |[本番]    |          |          |
| ...    |[打ち上げ] |          |[本番]    |          |
+--------+----------+----------+----------+----------+
         ↑ sticky left (時刻列)
```

### レイアウト定数

| 定数 | 値 | 意味 |
|------|-----|------|
| `H_START` | 10 | タイムライン開始時刻（10:00） |
| `H_END` | 23 | タイムライン終了時刻（23:00） |
| `PX_PER_MIN` | 2 | 1分あたりのピクセル数（1時間=120px） |
| `TOTAL_H` | 1560px | タイムライン全体の高さ |
| `TIME_COL_W` | 56px | 時刻ラベル列の幅 |
| `BAND_COL_W` | 160px | バンド列ひとつの幅 |
| `STAFF_COL_W` | 180px | スタッフ共通列の幅 |
| `HEADER_H` | 44px | ヘッダー行の高さ |
| `SNAP_MIN` | 10 | ドラッグスナップ単位（分） |

## データモデル

```typescript
type EventType = "rehearsal" | "performance" | "other";

interface TLEvent {
  id: string;
  bandId: string;       // バンドID or "__staff__"
  type: EventType;
  startMin: number;     // タイムライン開始時刻からの経過分 (0 = 10:00)
  durationMin: number;  // 持ち時間（分）
  note: string;         // メモ・表示テキスト
}
```

### 特殊 bandId

| 値 | 意味 |
|----|------|
| `"b1"` 〜 `"b8"` | 各バンドのID |
| `"__staff__"` | スタッフ共通列（常時表示、フィルター対象外） |

### イベント種別（バンド列のみ）

| type | 表示名 | 背景色 |
|------|--------|--------|
| `rehearsal` | リハーサル | 黄（#fef9c3） |
| `performance` | 本番 | 青（#dbeafe） |
| `other` | その他 | グレー（#f3f4f6） |

スタッフ共通列のイベントは種別を持たず、`note` フィールドをタイトルとして表示する。

## 列構成

### 時刻列（sticky left）
- 幅: `TIME_COL_W = 56px`
- 1時間ごとの時刻ラベルと :30 の補助ラベルを表示
- 水平スクロール時も固定

### スタッフ共通列
- 幅: `STAFF_COL_W = 180px`
- 常に表示（バンドフィルターの影響を受けない）
- 背景: 薄紫 (#f5f3ff)、左ボーダー: インジゴ (#6366f1)
- イベントは種別色なし・統一スタイル（紫左ボーダー + #ede9fe 背景）
- `note` をメインテキストとして表示

### バンド列（バンド別モード）
- 幅: `BAND_COL_W = 160px`
- バンドフィルターで個別表示/非表示を切り替え可
- 各バンドに固有カラー（8色パレット）

### 一列表示モード
- 全バンドのイベントを幅 320px の単一列に集約
- 同時刻に複数イベントが重なる場合は `assignSublanes()` で横分割
- イベントに バンド名 + 種別 を表示、左ボーダーでバンド識別

## インタラクション

### イベント追加
- 列の空きスペースをクリック → 10分スナップでその時刻をプリセットしてモーダルを開く
- スタッフ共通列クリック → モーダルはバンド選択・種別選択なしのシンプル構成

### イベント編集・削除
- イベントをクリック → 編集モーダル
- モーダル内「削除」ボタンで削除

### ドラッグ移動
- **縦ドラッグ**: 時間変更（10分スナップ）
- **横ドラッグ**: バンド列変更（`data-band-id` 属性を `document.elementFromPoint` で検出）
- 5px 未満の移動はクリックとして扱い、モーダルを開く（誤ドラッグ防止）

### バンド名変更
- ヘッダーのバンド名をダブルクリック → インライン編集

| フィールド | バンド列 | スタッフ列 |

## 叩きのタイムライン生成 (`genReverseRehearsal`)

予定がひとつもない状態でのみ押下可能。

### 生成ルール

| 区分 | 内容 |
|------|------|
| リハーサル | 出演順の**逆順**で、Band 8 から順に 10:00 スタート・20分枠 |
| 本番 | 出演順で 14:00 スタート・30分枠、35分間隔（転換5分含む） |
| 全体打ち合わせ | スタッフ共通・13:00〜13:30 |
| 開場 | スタッフ共通・13:30〜14:00 |
| 打ち上げ | スタッフ共通・19:00〜21:00（2時間） |

### 8バンド時の想定スケジュール

```
10:00 - 10:20  Band 8 リハーサル
10:20 - 10:40  Band 7 リハーサル
10:40 - 11:00  Band 6 リハーサル
11:00 - 11:20  Band 5 リハーサル
11:20 - 11:40  Band 4 リハーサル
11:40 - 12:00  Band 3 リハーサル
12:00 - 12:20  Band 2 リハーサル
12:20 - 12:40  Band 1 リハーサル

13:00 - 13:30  [スタッフ] 全体打ち合わせ
13:30 - 14:00  [スタッフ] 開場

14:00 - 14:30  Band 1 本番（1番手）
14:35 - 15:05  Band 2 本番（2番手）
15:10 - 15:40  Band 3 本番（3番手）

16:20 - 16:50  Band 5 本番（5番手）

```

## 主要アルゴリズム

### `assignSublanes(evts)`

一列表示モードで時間が重なるイベントを横に並べるためのサブレーン割り当て。

```
1. イベントを startMin 昇順でソート
2. 各イベントについて laneEnds[] から end <= startMin の最初のレーンを探す
3. 空きレーンがなければ新しいレーンを追加
4. laneEnds[lane] = startMin + durationMin を更新
5. 全イベント処理後、totalLanes = laneEnds.length でレーン数を確定
6. left = lane / totalLanes、width = 1 / totalLanes（比率）で配置
```
