# Requirements: ライブタイムテーブル管理

**Status**: Draft
**Created**: 2026-05-04
**Developer**: maru

## What（何を作るか）

ライブイベントに紐づく当日のタイムテーブルをDB永続化で管理する機能。
出演バンド（EventBand）ごとのレーンと、スタッフ共通タスク列を持つ縦型タイムラインビューを提供する。
ライブイベント詳細ページにタイムラインタブとして追加する。

## Why（なぜ必要か）

これまでスプレッドシートで管理していたが、視認性の悪さから情報共有に課題があったため。
アプリ内での一元管理により、関係者全員がリアルタイムで同一のスケジュールを確認できるようにする。

## Requirements（要件）

### Functional Requirements（機能要件）

- [ ] ライブイベントに紐づくタイムラインイベントをDB永続化して管理（CRUD）
- [ ] バンド列は出演バンド（EventBand）を参照して動的に表示
- [ ] スタッフ共通列でのタスク管理（開場・打ち上げ・全体打ち合わせ等）
- [ ] 叩きのタイムライン生成（逆リハ順での自動スケジュール生成）
- [ ] イベントのCRUD（クリック追加・編集・削除、ドラッグで時間/列変更）
- [ ] バンドフィルター・一列表示モード切り替え

### Non-Functional Requirements（非機能要件）

- [ ] 初期表示: 3秒以内（タイムライン全体のデータ取得含む）
- [ ] レスポンシブ対応（最小768px、スマホではスクロール操作で閲覧可）
- [ ] TypeScript 型安全性の確保（`any` 禁止）

## Success Criteria（成功基準）

- [ ] ライブイベントに紐づくタイムラインデータがDB保存・再読み込みできる
- [ ] 出演バンドが正しくタイムライン列として表示され、フィルター・一列表示が動作する
- [ ] 叩きのタイムライン生成・ドラッグ移動・モーダル編集が正常に動作する
- [ ] レスポンシブ表示でモバイルからも閲覧できる
- [ ] すべてのテストが通過する
- [ ] コードレビューで承認される
