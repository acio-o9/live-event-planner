# Requirements: Live Detail Tabs

**Status**: Draft
**Created**: 2026-04-30
**Developer**: maru

## What（何を作るか）

ライブ詳細画面（`/live-events/[id]`）にタブUIを追加し、「バンド一覧」「マイルストーン」「費用管理」の3セクションを個別タブで表示できるようにする。
バンド一覧・マイルストーンは既存コンポーネントをそのままタブ内に配置する。
費用管理は現状リンクのみだが、タブ選択時に費用管理コンポーネント（ExpenseList/ExpenseSummary）をインラインで表示する。

## Why（なぜ必要か）

ライブ詳細画面が縦長になってきており、情報が見づらい状態になっている。
タブで情報を整理することでUXを向上させる。

## Requirements（要件）

### Functional Requirements（機能要件）

- [ ] ライブ詳細画面に「バンド一覧」「マイルストーン」「費用管理」の3タブを実装する
- [ ] 「バンド一覧」タブ: 既存の参加バンド一覧UI（追加・編集・削除・メンバー管理）をそのまま表示
- [ ] 「マイルストーン」タブ: 既存の `MilestoneList` コンポーネントをそのまま表示
- [ ] 「費用管理」タブ: `ExpenseSummary` + `ExpenseList` + `ExpenseFormModal` をインライン表示（現状のリンクを廃止）
- [ ] アクティブタブはURLクエリパラメータ（`?tab=bands|milestones|expenses`）で状態保持し、ページリロード・リンク共有後も同じタブが選択された状態を維持する
- [ ] デフォルトタブは「バンド一覧」

### Non-Functional Requirements（非機能要件）

- [ ] タブ切り替えは即時（API呼び出しなし、クライアント側のみ）
- [ ] レスポンシブ対応（モバイルでもタブが正常に表示される）
- [ ] 既存の `AuthGuard` による認証制御を継続使用

## Success Criteria（成功基準）

- [ ] 3タブすべてが正常に切り替わり、各コンテンツが正しく表示される
- [ ] URLパラメータ変更でタブが切り替わり、直接URLアクセスで対応タブが表示される
- [ ] 費用管理タブで費用の追加・編集・削除・サマリー表示が動作する
- [ ] バンド一覧・マイルストーンの既存機能がタブ内でも正常に動作する（デグレなし）
- [ ] コードレビューで承認される
